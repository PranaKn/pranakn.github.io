  • Critical React2Shell RCE (CVE-2025-55182) Actively Exploited: The maximum-severity (CVSS 10.0) **React2Shell** flaw is an unauthenticated **Remote Code Execution** vulnerability in React Server Components' insecure deserialization logic (Flight protocol). It affects frameworks like Next.js, with over 77,000 IPs vulnerable and 30+ organizations already compromised by groups like UNC5174.   • IDEsaster Flaws in AI-Powered IDEs (e.g., Cursor, Copilot): Over 30 security flaws, collectively termed **IDEsaster**, including CVE-2025-49150, were disclosed in AI-powered IDEs. The issues chain prompt injection, auto-approved tool calls, and legitimate IDE features to achieve unauthorized **data exfiltration or RCE**. Attackers exploit auto-write capabilities to inject malicious settings into configuration files (like .vscode/settings.json) without user interaction. Developers must use AI agents only with trusted projects and manually review external context for hidden instructions. More info
  • Critical Apache Tika XXE Vulnerability (CVE-2025-66516): A critical **XML External Entity (XXE)** vulnerability (CVSS 10.0) in the **Apache Tika** content analysis toolkit (used by Elasticsearch, Solr) allows an unauthenticated attacker to embed a malicious **XFA file inside a PDF**. When Tika processes the PDF, the XXE is triggered in tika-core (versions 1.13 through 3.2.1), potentially allowing the attacker to read sensitive internal files, perform SSRF, or cause DoS. Users must immediately upgrade tika-core to **version 3.2.2** or later. More info
  • Zero-Click Agentic Browser Attacks Against Perplexity Comet: New zero-click attacks exploit the autonomy of agentic browsers granted OAuth access. The **"Google Drive Wiper"** attack uses a specially crafted email with natural language instructions (e.g., "delete loose files") that the agent interprets as legitimate housekeeping, leading to **mass file deletion without user confirmation**. Separately, the **HashJack** vulnerability forces AI browsers to execute hidden commands placed after the # symbol in legitimate URLs. Perplexity and Microsoft have patched affected browsers.   • ClayRat Android Spyware Escalates to Full Device Takeover: The **ClayRat** Android spyware has significantly escalated its capabilities by exploiting **Accessibility Services**. The updated version now records **lock screen credentials**, captures PINs/passwords, automatically unlocks the device, performs screen recording, and uses deceptive overlays to block user interaction and automatically disable Google Play Protect. Distributed via fraudulent phishing domains and Dropbox, ClayRat poses a severe risk to corporate BYOD environments through credential and data theft. More info
  • Novel Clickjacking Using SVG Filters and CSS: Security researcher Lyra Rebane devised a novel clickjacking technique that exploits a cross-origin information leak using **SVG filters (feBlend and feComposite)** and CSS. The SVG filters act as **logic gates** to process pixels from an underlying framed webpage, effectively turning SVG into a programmable device to simplify complex clickjacking attacks. A proof-of-concept exfiltrated text from Google Docs. Defense involves using the Intersection Observer v2 API to detect when an SVG filter covers an iframe. More info
  • Stealthy Malware Uses AutoIT3 FileInstall() for Shellcode Delivery: A wave of malware is exploiting the **AutoIT3 scripting language's FileInstall() function** to drop and execute hidden shellcode, enabling highly stealthy payload delivery. The function embeds the payload (like shellcode) directly within the compiled executable, drops it to %TEMP%, decrypts it using an obfuscated function, and uses Windows API calls (VirtualAlloc and DllCall with CallWindowProc) to launch the shellcode in memory, bypassing file-based detection. Observed samples delivered the **Quasar Remote Access Trojan (RAT)** and **Phantom stealer**. More info

  • FBI Warns of Spoofed IC3 Websites: The FBI has issued Public Service Announcement I-091925, warning that cybercriminals are actively **spoofing its Internet Crime Complaint Center (IC3)** website (ic3.gov) to steal sensitive victim data. Threat actors clone the official domain via phishing or SEO poisoning, promising to help victims report a crime or recover funds. The fake portals harvest PII, banking details, and other sensitive information. The FBI reminds users that the genuine IC3 never asks for payment, and the only safe way to submit a report is directly via **www.ic3.gov**. More info
  • MCP Sampling Attacks Hijack Coding Copilots: A new class of attack, dubbed **MCP Sampling Attack**, exploits the **Model Context Protocol (MCP)** sampling feature in coding copilots. Malicious servers leverage MCP to inject persistent, hidden instructions into the client. Proof-of-Concepts include **Resource Theft**, **Conversation Hijacking** (e.g., forcing "pirate speak"), and **Covert Tool Invocation** (e.g., writing a .tmpfile without consent). These hidden instructions modify the system without the user's knowledge, necessitating a **least-privilege model** for LLM tools and enhanced output sanitization. More info
  • Guide to Continuous Attack Surface Visibility: An article argues that relying solely on **passive internet-scan data** provides an incomplete and quickly stale view of the modern external attack surface due to cloud adoption and Shadow IT. To maintain an accurate defensive posture, organizations must implement **continuous, automated, active reconnaissance** that daily verifies external exposure, tracks changes, and validates findings in real-time. This active approach is necessary to reveal newly exposed services, misconfigurations, and rogue assets for timely remediation. More info
